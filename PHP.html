Laravel5 自定义路由页面无法访问

最近准备使用Laravel5框架搭建一个cms的项目.于是将Laravel源码下载安装好之后,就开始熟悉框架.首先是看看路由,控制器,模型在这个框架中应该怎么用.

遇到问题

按照文档中所说的,我自定义了一个路由,如下所示:

Route::get('/','WelcomeController@index');

Route::get('/contact', 'WelcomeController@contact');
然后打开浏览器,先测试根目录,输入localhost,可以正常显示页面,当输入localhost/contact的时候,就会显示该路由找不到.刚开始还以为是权限问题,后来证实不是.于是各种百度,各种谷歌,看到有人也遇到了这种问题,通过别人给的回复"rewrite 没有配置好"又回到文档,发现文档有这么一句话"Laravel 框架通过 public/.htaccess 文件来让网址中不需要 index.php。如果你的服务器是使用 Apache，请确认是否有开启 mod_rewrite 模块".

解决问题

开启apache的rewrite模块

在终端输入命令

sudo a2enmod rewrite
打开/etc/apache2/mods-enabled/这个目录就会发现目录下多了一个

rewrite.load -> ../mods-available/rewrite.load
这就说明rewrite服务已经开启了。

然后打开/etc/apache2/apache2.conf文件里，在配置localhost的xml标签下，添上


Options FollowSymLinks MultiViews
AllowOverride All
Order allow,deny
allow from all

这里的路径按照自己的情况填写.输入service apache2 restart命令重启apache,如果还不能访问,则查看自己的localhost的绝对路径是否填写正确.

关于rewrite拓展

rewrite的主要功能就是实线url的跳转，它的正则表达式是基于perl语言。而且实现有两种方式，一种是在服务器那里写规则，还有一种就是在具体项目文件里面写.htacess文件来写规则

laravel框架用的是后者，而且public文件夹下就已经配好了.htacess文件，所以只要服务器开启了rewrite功能，还有指明哪个路径允许重写就行.

以下是laravel框架里的.htacess文件的rewrite配置

Redirect Trailing Slashes If Not A Folder...

RewriteCond %{REQUEST_FILENAME} !-d

RewriteRule .*/\( /\)1 [L,R=301]

Handle Front Controller...

RewriteCond %{REQUEST_FILENAME} !-d #请求的路径不存在
RewriteCond %{REQUEST_FILENAME} !-f #请求的文件不存在
RewriteRule ^ index.php [L] #满足上述条件则跳转到index.php
上述的规则表示，请求的文件或路径不存在则跳转到index.php，比如说请求的是localhost/contact,这个在文件系统中是找不到的，于是就跳转到localhost/index.php这个文件。然而获取$_SERVER['REQUEST_URI']里后面的contact这个参数，交给route处理。

也就说，主要输入的路径在实际的文件系统中找不到，就一律跳转到index.php,至于后面具体的哪个页面，则交给route来判断。

能够解决这个问题,还要感谢May's Blog

PHP与MySQL（总结篇）

其实之前写过关于PHP中MySQL应用的总结，不过那是在使用thinkPHP框架写项目的时候总结的，当时还没有系统的学习数据库，还是很懵懂。现在想想当时写的总结，原来只是应用于thinkPHP中的MySQL函数，这次系统的学习PHP开发，才发现和MySQL直接交互的PHP操作并不是那些函数，。现在我就来做些总结吧。其实，只要掌握了要点，也没有多少内容。

基本函数

建立数据库连接：mysql_connect();该函数的参数有三个，主机地址，用户名，用户密码

关闭数据库连接：mysql_close();该函数有一个参数，那就是来连接标识符

其实，数据库连接是短暂性连接，用完之后会自己断开，可是为什么还需要关闭数据库连接函数呢？因为：存在必是合理。。。。

选择使用的数据库：mysql_select_db();该函数的参数是要是用的数据库名

执行sql语句：mysql_query();该函数的参数就是一个string类型的数据，是要执行的正确的sql语句。

选择字符集：mysql_query("set names utf8");即选择utf8字符集。

查看错误：mysql_error();该函数没有参数，一般就是在发现操作不成功的时候调用

插入数据：mysql_query(sql插入语句);

查看插入id：mysql_insert_id();用于插入操作之后，返回新插入的数据的id（必须得有自增id才可以哦），如果插入多条数据，则返回第一个插入的数据的id。此函数可以判断是否成功插入，因为如果不成功，id会为0哦。

查看受影响的记录数目：mysql_affected_rows();传递的参数是连接标识符

header("Content-type:text/html;charset=utf-8");
//连接数据库
if(\(con = mysql_connect("localhost",'root','')) echo "连接成功,返回true\n";
else echo "连接失败，返回false\n";
//选择数据库
if(mysql_select_db('test')) echo "数据库选择成功，返回true\n";
else echo "数据库选择失败，返回false\n";
//指定数据操作的编码
mysql_query('set names utf8');
//通过变量向数据库中插入数据

\(sql = "insert into user(name) values('\)name')";//变量的单引号不能省略

echo "插入数据失败";
else
echo "插入数据成功，id为".\)re_id;
//获取本次操作受影响的数据条数
echo "受影响的数据条数为：".mysql_affected_rows(\(con);
if(mysql_close(\)con)) echo "数据库关闭成功，返回true";
else echo "数据库关闭失败，返回false";
数据查询

查询语句：mysql_query("select 查询语句");该语句会返回一个资源句柄，可以通过该资源句柄获取到结果集中的数据

获取数据的四种方法

mysql_fetch_row()：该函数能够接收一个参数为资源标识符，标志着查询出来的结果所存放的位置，每执行一次，都会从结果集中依次取出一条数据，并以数组的形式返回出来，当所有的数据都取出来之后，再取数据的时候会返回空。返回的数组是一维索引数组，每一个下标与数据表中字段的位置相对应。

Mysql_fetch_array():该函数的作用和mysql_fetch_row()是一样的，只不过该函数再返回的数据中除了有mysql_fetch_row()包含的数据外，还包含了一组关联数组，即字段名对应着字段值，这些数据以一维数组的形式展现。这个函数的好处就是可以根据字段名获取相应的字段值，而不需要知道每一个字段所在的位置，不过速度会很慢。该函数还有一个参数，该参数有3种形式：MYSQL_ASSOC只输出关联数组；MYSQL_NUM只输出数字索引数组；MYSQL_BOTH输出关联数组和索引数组，这也是默认状态。

mysql_fetch_assoc()以关联数组形式获取数据，该函数和mysql_fetch_array('资源定位符',MYSQL_ASSOC)效果是一样的

mysql_fetch_object()函数是从结果集中获取数据，然后以对象的形式打印出来,可以获取到对象中的任意一个属性。

获取结果集中行的数目：mysql_num_rows();可以通过该方法判断是否查询到数据

获取某一个值：mysql_result();该函数获取结果集中某一个字段的值，有三个参数，第一个参数是结果集的地址，第二个参数是行号（从0开始）,第三个参数是获取哪一个字段，默认为第一个字段（从0开始）。一般会用在直接取某一行的某一个字段，而不是循环显示的时候。

//mysql_fetch_row()函数查询数据并打印
\(res = mysql_query("select id,name,num from user");
while(\)row1 = mysql_fetch_row(\(res)){print_r(\)row1);echo "
";}
//mysql_fetch_array()函数只有一个参数的查询数据并打印
\(res = mysql_query("select id,name,num from user");
while(\)row1 = mysql_fetch_array(\(res)){print_r(\)row1);echo "
";}
//mysql_fetch_array()函数有两个参数的查询数据并打印
\(res = mysql_query("select id,name,num from user");
while(\)row1 = mysql_fetch_array(\(res,MYSQL_ASSOC)){print_r(\)row1);echo "
";}
//mysql_fetch_assoc()以关联数组形式获取数据
\(res = mysql_query("select id,name,num from user");
while(\)row1 = mysql_fetch_assoc(\(res)){print_r(\)row1);echo "
";}
//mysql_fetch_object()函数是从结果集中获取数据，然后以对象的形式打印出来,可以获取到对象中的任意一个属性
\(res = mysql_query("select id,name,num from user");
while(\)row1 = mysql_fetch_object(\(res)){print_r(\)row1);echo \(row1->name."<br/>";}
//获取结果集的行数
\)res = mysql_query("select id,name,num from user");
echo mysql_num_rows(\(res);
//mysql_result()获取结果集中某一个字段的值
\)res = mysql_query("select id,name,num from user");
echo mysql_result($res,2,1);
其实,其实,最后再加一句,PHP升级之后,这些操作mysql的一些函数已经过时了,取而代之的是mysqli扩展库.不过这些基础性的东西看一看也是没有坏处的.

ThinkPHP奇葩现象:数据表新增字段无法存储数据

之前写的一个已经发布的网站中,需要根据用户登录账号判断投稿者,于是就在数据表中添加了一个字段,无奈在本机可以正常获取数据,但是上传到网站上却不能获取.

通过一步步的排查问题发现,从用户输入的数据可以正常获取,但是存储数据库的时候却不能够将新添加的字段数据更新进去,因为更新的SqL语句中根本没有这一项数据.

感觉真的是好奇怪,本来还以为apache出了问题,重启了一下,还是不能解决.于是在网上百度了以下这个状况,还真有前辈遇到了此问题并成功解决.

文章中提到这是ThinkPHP缓存文件Runtime导致的,将这个文件整个都删除就可以了.删除语句为:rm -fr Runtime/ 就成功解决了此问题,而删除语句中-fr参数的具体用法详见此文章:http://yanzl.net/1122.html.

其中,有一个现象和参考文章作者遇到的现象是一样的,那就是在本机运行的时候,开启的是开发模式,即:Debug＝true,而上传到网站上之后,开启的是运行模式,即:Debug＝false,在运行模式下,运行缓存文件Runtime没有得到更新,所以该程序不认为又新添加了字段,故没有将该字段的数据存入数据库,把缓存文件删除之后,在此更新网站,会产生新的缓存文件,于是就将问题解决了.

现在献上解决问题的参考文章:http://www.xcoder.cn/index.php/archives/1100

新增和修改比较

新增是在没有的基础上增加新的数据,不需要在数据库中查找信息,调用的是add()方法

修改是在原来已经有数据的基础上进行修改,所以传递的所有信息有包含了不变的id,即保证id不能够被修改。

php实现后台登陆功能

最近想借助thinkphp框架写一个关于图书管理的小项目，经过一天对整个项目进行整体的规划之后决定先写后台。

今天用php写了关于后台登陆功能，其中主要涉及到了从前端接收数据，在后台判断用户名和密码是否正确，刷新后台的时候自动判断用户是否已经登陆以及退出功能。

接受数据用的I方法

I方法值input（输入），用于更加方便和安全的获取系统输入的变量，用法格式为:

I('变量类型.变量名',['默认值'],['过滤方法'])

变量类型指post，get等请求方式或者输入类型，变量类型不区分大小写

默认值和过滤方法属于可选参数，如果写过滤方法，就一定要写默认值，可以为空。

I方法支持获取整个变量类型，比如：I('get');//获取整个get数组内容。

I方法中param是特殊的，它可以自动判断当前请求类型的变量获取方式，比如：I('param.id');等同于：I('id');

所以，可以使用I方法获取从前端传输进来的数据：

\(username = I('username');
\)pas = I('password');
用户信息验证

提取到用户登录数据之后，在数据库中查找用户信息，看用户名和密码是否匹配。

\(user = M('users')->where(array('user_name'=>\)username))->find();
if(!\(user || \)user['user_password']!=$pas)
{
echo "用户名或密码错误";
}
往session中写入数据

在经过用户验证成功之后，就需要把用户信息写入session中，以减少客户端访问数据库的频率。

Session 的工作机制是：为每个访客创建一个唯一的 id (UID)，并基于这个 UID 来存储变量。UID 存储在 cookie 中，或者通过 URL 进行传导。

//往session中写入数据
session('uid',\(user['user_id']);
session('username',\)user['user_name']);
session('userpwd',$user['user_password']);
从登录页面跳转到后台首页面

$this->redirect('Admin/Index/index');
判断用户是否登陆

在刷新后台页面或者进入后台的一个页面的过程中，我们都要判断用户是否已经登陆，因为如果用户没有登陆，就只能访问前端而不能访问后台，所以判断用户是否登陆是所有后台都需要执行的功能，为了方便，我们可以在要进入该页面之前自动调用判断方法，那么就需要有一个公共的控制器，然后后台所有页面的控制器都继承这个公共的控制器。

新建控制器名为：CommonAction.class.php，在该控制器中写一个可以自动调用的方法。如果用户没有登陆，那么自动跳转到后台登录页面。

<?php

Class CommonAction extends Action{
public function initialize(){
if(!isset($SESSION['uid'])||!isset(\(_SESSION['username'])){
\)this->redirect('Admin/Login/index');
}
}
}
?>
退出登陆

在检测是否已经登陆的时候，我们是检测session中有没有数据，如果要退出登陆的话，就直接清楚session中的所有数据就可以了。

为后台首页退出按钮添加链接，即触发此事件之后就交由控制器的相应方法进行处理。

退出

session_unset();

\(this->redirect('Admin/Login/index');
}
//session_unset()释放当前在内存中已经创建的所有\)_SESSION变量，但不删除session文件以及不释放对应的session id

// session_destroy()删除当前用户对应的session文件以及释放session id，内存中的$_SESSION变量内容依然保留,因此，释放用户的session所有资源

在所有数据都清空之后，自动转到后台登录页面。
