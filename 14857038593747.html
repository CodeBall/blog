MySQL学习笔记（3）——修改数据表及写数据

2016/1/21 posted in  数据库

操作字段

添加单个字段：

alter table 表名 add [column] 字段名称 字段定义 [first|after 字段名称]
first指明将该字段添加到表定义的最前端，after指明将该字段添加到某个字段的后面，默认情况下是添加到最后。

添加多个字段：

alter table 表名 add column
添加多个字段是不能够指定位置的，很显然，没法指定。。。

删除一个字段：

alter table 表名 drop 字段名称
删除多个字段：

alter table 表名 drop 字段名称,drop 字段名称
删除字段的同时添加字段：

alter table 表名 drop 字段名称,drop 字段名称,add 字段名称 字段定义
操作约束

添加主键约束：

alter table 表名 [constraint [约束名称]] primary key 约束类型
添加唯一约束：

alter table 表名 [constraint [约束名称]] unique key 约束类型
添加外键约束：

alter table 表名 [constraint [约束名称]] foreign key (外键字段名) references 从表名（字段）
添加|删除默认约束：

alter table 表名 later [column] 字段名 {set default 默认值|drop default}
删除主键约束：

alter table 表名 drop primary key
删除唯一约束：

alter table 表名 drop {index|key} 约束名
删除外键约束：

alter table 表名 drop foreign key 约束名
操作数据表

修改字段定义：

alter table 表名 modify [column] 字段名 字段定义 [first|after 字段名]
修改字段名称：

alter table 表名 change [column] 旧字段名称 新字段名称 字段定义 [first|after 字段名]
修改数据表名称：

alter table 表名 rename [to|as] 新名称
rename table 表名 to 新名 [,表名1 to 新名1]...
插入数据

方法一

插入多条数据

insert [into] 表名 [(字段名称)] values(字段值)
字段名称可写可不写，如果不写，则字段值必须包含所有字段的值，如果字段是自动编号，则可以写为null或者default；如果写字段名，则按照给定的字段顺序依次填入相应的字段值，字段值可以是表达式，如果该字段有默认值，也可以是默认值（default）

插入多条数据

insert [into] 表名 [(字段名称)] values(字段值1),(字段值2),...
如果插入多条数据，则必须有相同个字段，即不能第一条数据有3个值，第二条数据有5个值。

方法二

insert 表名 set 字段名1 = '字段值1',字段名2 = '字段值2',...
这种方法只能插入一条语句，可以进行子查询，具体来说是比较运算符=引发的子查询

方法三

insert [into] 表名 [(字段名称)] select 选择语句
这种方法借助查询语句来从另外一张表中查找到相应数据插入到该表中

比如在产品表中查询产品的分类，将查询的结果写入到分类表中

insert goods_cates (cate_name) select
goods_cate from goods group by goods_cate;
修改数据

单表修改

update 表名 set 字段名=’更新的该字段的值’[,字段名1=’更新的该字段的值1’,...] [where 更新条件]
如果不写更新条件，则所有的数据都将更新

多表修改

多表更新涉及到表与表之间的连接，根据一张表的内容更新另一张表的内容

例如：根据产品类型表(goods_cate)更新产品表(goods)中的产品类型

Update goods inner join goods_cate on goods_cate = cate_name set goods_cate = cate_id;
删除数据

单表删除

Delete from 表名 [where 删除条件]
如果不写删除条件，则该表中所有的数据都会被删除。

多表删除

所谓的多表删除，就是通过多张表的连接来确定删除的内容。

例如：在一张表(goods)中，有很多重复的记录，如果想删除重复的记录，不可能找到它们的id一个一个的删除，这就用到了多表删除。把这张表和由重复数据组成的虚拟的表进行连接，进而进行删除操作。

Delete t1 from goods as t1 left join(
select goods_id,goods_name from goods group by goods_name having count(goods_name)>=2) as t2
on t1.goods_name = t2.goods_name
where t1.goods_id > t2.goods_id;
