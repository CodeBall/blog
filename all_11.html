ThinkPHP奇葩现象:数据表新增字段无法存储数据

之前写的一个已经发布的网站中,需要根据用户登录账号判断投稿者,于是就在数据表中添加了一个字段,无奈在本机可以正常获取数据,但是上传到网站上却不能获取.

通过一步步的排查问题发现,从用户输入的数据可以正常获取,但是存储数据库的时候却不能够将新添加的字段数据更新进去,因为更新的SqL语句中根本没有这一项数据.

感觉真的是好奇怪,本来还以为apache出了问题,重启了一下,还是不能解决.于是在网上百度了以下这个状况,还真有前辈遇到了此问题并成功解决.

文章中提到这是ThinkPHP缓存文件Runtime导致的,将这个文件整个都删除就可以了.删除语句为:rm -fr Runtime/ 就成功解决了此问题,而删除语句中-fr参数的具体用法详见此文章:http://yanzl.net/1122.html.

其中,有一个现象和参考文章作者遇到的现象是一样的,那就是在本机运行的时候,开启的是开发模式,即:Debug＝true,而上传到网站上之后,开启的是运行模式,即:Debug＝false,在运行模式下,运行缓存文件Runtime没有得到更新,所以该程序不认为又新添加了字段,故没有将该字段的数据存入数据库,把缓存文件删除之后,在此更新网站,会产生新的缓存文件,于是就将问题解决了.

现在献上解决问题的参考文章:http://www.xcoder.cn/index.php/archives/1100

Python count()方法\replace()方法

count()方法

函数原型

str.count(sub, start= 0,end=len(string))
函数描述

count() 方法用于统计字符串里某个字符出现的次数。可选参数start和end为在字符串搜索的开始与结束位置。

参数介绍

sub -- 搜索的子字符串
start -- 字符串开始搜索的位置。默认为第一个字符,第一个字符索引值为0。
end -- 字符串中结束搜索的位置。字符中第一个字符的索引为 0。默认为字符串的最后一个位置。
返回值

该方法返回子字符串在字符串中出现的次数。

示例

str = "this is a string example"
sub = 'exa'
print "str.count('is',0,len(str)):",str.count('is',0,len(str))
print "str.count('s'):",str.count('s')
print "str.count(sub):",str.count(sub)

运行结果如下
str.count('is',0,len(str)): 2
str.count('s'): 3
str.count(sub): 1

replace()方法

函数原型

str.replace(old, new[, max])
函数描述

replace() 方法把字符串中的 old（旧字符串） 替换成 new(新字符串)，如果指定第三个参数max，则替换不超过 max 次。

参数介绍

old -- 将被替换的子字符串。
new -- 新字符串，用于替换old子字符串。
max -- 可选字符串, 替换不超过 max 次
返回值

返回字符串中的 old（旧字符串） 替换成 new(新字符串)后生成的新字符串，如果指定第三个参数max，则替换不超过 max 次。

示例

str = "this is string example....wow!!! this is really string";
print str.replace("is", "was");
print str.replace("is", "was", 3);

执行结果为:
thwas was string example....wow!!! thwas was really string
thwas was string example....wow!!! thwas is really string

Python capitalize()方法

函数原型

str.capitalize()
函数描述

Python capitalize()将字符串的第一个字母变成大写,其他字母变小写。对于 8 位字节编码需要根据本地环境.

返回值

该方法返回一个首字母大写的字符串,并且只有首字母是大写,其他字符都是小写。

示例

str = "this is a String!"
print "str.capitalize():", str.capitalize()
a = 'wORD'
print "a.capitalize():",a.capitalize()

运行结果为:
str.capitalize(): This is a string!
a.capitalize(): Word

Python urllib库

urlopen()方法

函数原型

urllib.urlopen(url[, data[, proxies]]) ;

参数描述

创建一个表示远程url的类文件对象，然后像本地文件一样操作这个类文件对象来获取远程数据。

参数url表示远程数据的路径， 一般是网址；

参数data表示以post方式提交到url的数据(玩过web的人应该知道提交数据的两种方式：post与get。

参数proxies用于设置代理。

urlopen返回 一个类文件对象，他提供了如下方法： read() , readline() , readlines() , fileno() , close() ：这些方法的使用方式与文件对象完全一样;

info()：返回一个httplib.HTTPMessage 对象，表示远程服务器返回的头信息；

getcode()：返回Http状态码。如果是http请求，200表示请求成功完成;404表示网址未找到；

geturl()：返回请求的url；

示例

WORD_URL = "http://learncodethehardway.org/words.txt"
WORDS = []
for word in urlopen(WORD_URL).readlines():
WORDS.append(word.strip())

print urlopen(WORD_URL).info()
print urlopen(WORD_URL).getcode()
print urlopen(WORD_URL).geturl()

运行之后返回以下结果

urllib中一些辅助方法用于对url进行编码、解码

url中是不能出现一些特殊的符号的，有些符号有特殊的用途。
当以get方式提交数据的时候，会在url中添加key=value这样的字符串，
所以在value中是不允许有'='，因此要对其进行编码；与此同时服务器接收到这些参数的时候，要进行解码，
还原成原始的数据。这个时候，这些辅助方法会很有用：
urllib.quote(string[, safe])：对字符串进行编码,参数safe指定了不需要编码的字符;
urllib.unquote(string) ：对字符串进行解码；
urllib.quote_plus(string [ , safe ] ) ：与urllib.quote类似，但这个方法用'+'来替换' '，
而quote用'%20'来代替' '
urllib.unquote_plus(string ) ：对字符串进行解码；
urllib.urlencode(query[, doseq])：将dict或者包含两个元素的元组列表转换成url参数。
例如 字典{'name': 'dark-bull', 'age': 200}将被转换为"name=dark-bull&age=200" urllib.pathname2url(path)：将本地路径转换成url路径； urllib.url2pathname(path)：将url路径转换成本地路径；

Python strip()函数介绍

方法功能

strip()方法用于移除字符串头尾指定的字符

函数原型

str.strip(ch) 删除str字符串中开头,结尾处,位于ch删除序列的字符

str.lstrip(ch) 删除str字符串中开头处,位于ch删除序列的字符

str.rstrip(ch) 删除str字符串中结尾处,位于ch删除序列的字符

返回值

返回移除过之后的字符串

示例

str = "000this is a string\texample,let's look if carefuly!0000"
print "注意:只有两端的才有效果,中间的制表符是没有效果的"
print str.strip('0')
print str.lstrip('0')
print str.rstrip('0')

该段代码运行结果为:
this is a string example,let's look if carefuly!
this is a string example,let's look if carefuly!0000
000this is a string example,let's look if carefuly!
注意:

1.如果ch为空的话,默认删除空白符（包括'\n', '\r', '\t', ' ')

a = ' 123'
b = '\tabcd\t'
print a.strip()
print b.strip()

该段代码的运行结果为:
123
abcd

可见之前的空格和制表符的效果都消失了
2.这里的rm删除序列是只要边（开头或结尾）上的字符在删除序列内，就删除掉,不管顺序是怎么样的

c = "123abc"
print c.strip('21')
print c.strip('12')

这段代码的运行结果为:
3abc
3abc

可见两次处理结果是一样的
