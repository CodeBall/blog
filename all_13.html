图的基本概念及图的存储

各种基本概念

1.图：顶点集合和顶点间的二元关系集合组成的数据结构，用G（V,E）表示。其中顶点集合用V(G)表示，边的集合用E(G)表示。

2.无向图：图中所有的边都没有方向性的图成为无向图。

3.有向图：图中所有的边都有方向性的图成为有向图。

4.边和点：表示一条从u到v的又向边，其中u是这条有向边的起点，v是这条有向边的终点。

5.基图：忽略有向图的所有边的方向，得到的图称为该有向图的基图。

6.完全图：如果无向图中任何一对顶点之间都有一条边，这种无向图称为完全图。

7.有向完全图：如果有向图中任何一对顶点之间都有一条边，这种有向图称为有向完全图。

8.稀疏图：边或弧的数目相对较少的图称为稀疏图。

9.稠密图：边或弧的数目相对较多的图称为稀疏图。

10.平凡图：只有一个顶点的图称为平凡图，平凡图的阶为1.

11.零图：边的集合为空的图成为零图。

12.度：一个顶点的度数是与它相关联的边的数目，记作deg(u)。在有向图中，顶点的度数等于该顶点的入度和出度的和。某一个顶点的出度是以该顶点为起点的有向边的数目，记作od(u)，某个顶点的入度是以该顶点为终点的有向边的数目，记作id(u)。

拓展：在无向图和有向图中，所有顶点度数总和等于边数的两倍。

13.奇点和偶点：度数为基数的顶点为奇点，度数为偶数的顶点为偶点。

14.孤立顶点：度数为0的顶点称为孤立顶点。

15.路径：从一个点出发，经过一系列的点到达另一个点的通路称为两点之间的路径（路径可以包含很多点）。

16.生成树：一个无向联通图的生成树是它的包含所有顶点的极小连通子图，这里所谓的极小指边的数目极小。

17.连通分量：如果一个无向图不是连通的，则其极大连通子图成为连通分量。这里所谓的极大指子图中包含的顶点个数极大。

18.强连通图：对于图中每一对顶点，既存在从u到v的路径，也存在从v到u的路径，则称此图为强连通图。

19.强连通分量：对于非强连通图，其极大强连通子图称为其强连通分量。

图的存储表示

1.邻接矩阵

设G(V,E)是一个具有n个顶点的图，矩阵是一个n*n的二维数组，用Edge[i][j]表示。

i,j分别代表一条边的两个端点，如果两个端点之间有直接相连的边，则Edge[i][j]存储为1，否则存储为0.

注意：邻接矩阵不能够表示以下形式：

（1）图中存在自身环。

（2）多条边的起点一样，终点也一样，称为重边。

示例：每行输入两个数，代表两点之间有边存在，顶点编号最大为100.

描述代码为：

int Edge[101][101];
    int n,m,i,j,u,v;
    
    while(scanf("%d%d",&n,&m))//n个点，m条边
    {
        memset(Edge,0,sizeof(Edge));
        if(n == 0 && m == 0)
            break;
        for(i = 0;i < m;i++)
        {
            scanf("%d%d",&u,&v);//读入边的起点和终点
            Edge[u][v] = 1;//标记该边已存在
        }
        for(i = 1;i <= n;i++)//计算每个顶点的出度
        {
            int od = 0;
            for(j = 1;j <= n;j++)
            {
                od += Edge[i][j];
            }
            printf("第%d个顶点的出度为%d\n",i,od);
        }
//用相似的方法计算每个顶点的入度，固定j,遍历i即可。
    }

2.邻接表

如上图所示，最右边的矩阵是最左边的图对应的邻接矩阵

中间是最左边的图对应的邻接表。可以看出，邻接表是一个个链表的形式，前面是一个头结点，代表没一个顶点，后面是一条条的链表，代表该顶点相关联的边的另一个顶点。

邻接表的结构体声明：

#define MAXN 100
struct ArcNode//边结点
{
    int adjvex;//有向边的另一个邻接点的序号
    ArcNode *nextarc;//指向下一个边结点的指针
};
struct VNode//顶点
{
    int data;//顶点信息
    ArcNode *head1;//出边表的表头指针
    ArcNode *head2;//入边表的表头指针
};
struct LGraph//图的邻接表存储结构
{
    VNode vertexs[MAXN];//顶点数组
    int vexnum , arcnum;//顶点数目和边的数目
};
LGraph lg;

建立邻接表的基本操作为：

void CreateLG()
{
    int i = 0;
    ArcNode *pi;
    int v1,v2;
    lg.vexnum = lg.arcnum = 0;
    scanf("%d%d",&lg.vexnum,&lg.arcnum);
    for(i = 0;i < lg.vexnum;i++)
    {
        lg.vertexs[i].head1 = lg.vertexs[i].head2 = NULL;
    }
    for(i = 0;i < lg.arcnum;i++)
    {
        scanf("%d%d",&v1,&v2);
        pi = new ArcNode;
        pi->adjvex = v2;
        pi->nextarc = lg.vertexs[v1].head1;
        lg.vertexs[v1].head1 = pi;
        pi = new ArcNode;
        pi->adjvex = v1;
        pi->nextarc = lg.vertexs[v2].head2;
        lg.vertexs[v2].head2 = pi;
    }
}

存储分析

1.邻接表的时间复杂度为O（m）或O(2m)（无向图），邻接矩阵的时间复杂度为O(n2)。

2.邻接表的空间复杂度为O(n+ m）或O(n + 2 * m)；邻接矩阵的空间复杂度为O(n * n);

字典树

定义

字典树又称单词查找树，Trie树，是一种如图1所示的树形结构。

结构

每棵字典树都有一个根节点，每一个结点都有26个指针，分别指向a到z26个字母，有一个数据域，存储要记录的数据，有一个标记域，标记字符串是否结束。

struct Trie
{
    //因为next与字符有一一对应的关系，所以不需要存储数据
    int sign;//标记字符串结尾
    struct Trie *next[26];//指向下一个字符
}*head,*p,*now;
开辟新的存储空间

Trie *new()
{
    Trie *p = new Trie;//开辟内存
    for(i = 0;i < 26;i++)//初始化操作
    {
        p->next[i] = NULL;
    }
    p->sign = 0;
    return p;
}

字典树的初始化

下面的代码只是添加一个字符串，如果添加多个字符串，可以在调用函数中进行修改，也可以在下面的函数中加循环进行控制。

void init()
{
    char sz[30];
    int i,j,k,l;
    scanf("%s",sz);
    l = strlen(sz);
    head = new();
    p = head;
    for(i = 0;i < l;i++)
    {
        j = sz[i] - 'a';
        if(p->next[j] == NULL)
        {
            now = new();
            now->data = sz[i];
            p->next[j] = now;//将新建的节点连接在字典树上
        }
            p = p->next[j];
    }
    p->sign = 1;
}

字符串的查找

给定一个字符串，询问该字符串是否已经出现过，即在原来建立的字典树上进行查找的过程。

字符串查找的过程和建树的过程很相似，只不过需要判断节点是不是字符串的结束。

bool lookup()
{
    char sz[30];
    int i,j,l;
    scanf("%s",sz);
    p = head;
    l = strlen(sz);
    for(i = 0;i < l;i++)
    {
        j = sz[i] - 'a';
        //没有这个字符，则下面都没有，返回失败
        if(p->next[j] == NULL)
            return false;
        else
            p = p->next[j];
        //如果已是要查询的字符串最后一个字符，判断有没有结束标志
        if(i == l - 1)
        {
            if(p->sign == 1)
                return true;
            else
                return false;
        }
    }
}

释放内存

所有的这一组数据使用完之后，就要对其进行释放内存。

void delet(Trie *head)
{
    if(head)
    {
        for(int i = 0;i < 26;i++)
        {
            if(head->next[i]!=NULL)
                delet(head->next[i]);
        }
    }
    delete head;
    head = NULL;
}

字典树的性质

（1）根节点不包含字符，除根节点外每一个结点都只包含一个字符

（2）从根节点到某一个节点，路径上的节点连接起来就是一个字符串

（3）每个节点的所有子节点包含的字符都不相同

字典树的应用

（1）串的检索

给出一些已有的单词作为熟词，再给出一些词汇，找出这些词汇中没有在熟词里出现的词。

可以用枚举，哈希，字典树进行搜索

（2）“串”的排序

给出一些互不相同的单词，按照字典序进行排列

采用数组的方式创建字典树，这棵树的每个结点的儿子按照字母大小排序，再对这棵树进行先序遍历。

（3）最长公共前缀

查找任意两个字符串的最长公共前缀

对所有串建立字典树，对于两个串的最长公共前缀的长度即他们所在的结点的公共祖先个数，于是，问题就转化为当时公共祖先问题。

哈希表

概念

为了方便查找，在存储位置和关键字之间建立一个确定的关系，当查找关键字的时候，根据建立的关系对存储位置进行访问，有效地减少了多次比较带来的时间的浪费。这个关系称为哈希函数，这种思想称为哈希思想。

哈希函数是一个映像，可以灵活得定义。但是，对于不同的关键字，经过相同的哈希函数的处理后，有可能得到相同的地址，这种现象称为冲突，并且，一般情况下，冲突只能尽可能的减少，而不能完全避免。

哈希函数的构造方法

（1）直接定址法

去关键字或关键字的某个线性函数作为哈希函数，由此计算哈希地址的方法。

H(key) = key或H(key) = a*key + b(a,b为常数)。

（2）数字分析法

假设关键字是R进制数，并且哈希表中可能出现的关键字都是事先知道的，则可取关键字的若干数位组成哈希地址。

（3）平方取中法

取关键字平方后的中间几位作为哈希地址。

（4）折叠法

将关键字分割成位数相同的几部分（最后一部分的位数可以不同），然后取这几部分的叠加和作为哈希地址的方法。

适用于关键字位数很多，而且关键字中每一位上数字分布大致均匀。

（5）除留余数法（最简单，最常用）

取关键字被某个不大于哈希表表长m的数p除后所得余数为哈希地址。H（key） = key % p;

在选取p的时候，一般选质数或者不包含小于20的质因数的和数。

（6）随机数法

选择一个随机函数，取关键字的随机函数值为它的哈希地址。

冲突的处理

（1）开放定址法

Hi = （H（key）+di）%m;i = 1,2,...,k（k <= m-1）

H（key）为哈希函数，m为哈希表表长，di为递增序列

di有三种取法

线性探测再散列：di = 1,2,3,...,m-1;保证只要哈希表未填满，总能找到一个不发生冲突的地址

二次探测再散列：di = 11,-1-1,22,-2-2,33,...,kk,-k-k;只有在哈希表长满足4j+3一形式的素数时才会将冲突处理好。

伪随机探测再散列：di = 伪随机数序列。

二次聚集 在处理冲突过程中发生的两个第一个哈希地址不同的记录争夺同一个后记哈希地址的现象，即在处理同义词的冲突过程中又添加了非同义词的冲突。

（2）再哈希表

Hi = R*Hi(key)，i = 1,2,3。。。k;

R*Hi(key)均是不相同的哈希函数，即在同义词产生地址冲突时计算另一个哈希函数，直到不产生地址冲突为止。

（3）链地址法（方便，常用）

将所有关键字为同义词的记录存储在同一线性链表中。假设某哈希函数产生的哈希地址在区间【0,m-1】中，则建立一个头链表。每个链结的初始状态都是空指针，当有关键字对应一个链结的时候，在该链结后加关键字，如图

（4）建立一个公共溢出区

一个基本表，每个分量存放一个记录，另设一个溢出表，所有关键字和基本表中关键字为同义词的记录，不管哈希地址是什么，一旦发生冲突，都填入溢出表。

哈希表的查找

给定K值，根据建表时设定的哈希函数求得哈希地址，若表中此位置上没有记录，则查找不成功，否则比较关键字，如果给定的K和表中记录相等，则查找成功；否则根据建表时设定的冲突处理的方法进行下一个地址的查找，直到哈希表的某个位置为空或者找到为止。

查找过程中需和给定值进行比较的关键字的个数取决于三个因素：哈希函数，处理冲突的方法和哈希表的装填因子。

装填因子：a = （表中填入的记录数）/（哈希表的长度）。标志哈希表的装满程度。

平衡二叉树

基本概念

平衡二叉树又称AVL树，它的左子树和右子树都是平衡二叉树，并且左子树和右子树的深度之差的绝对值不超过1.

平衡因子：该结点的左子树的深度减去该结点的右子树的深度，所以平衡二叉树上的所有结点的平衡因子只可能是-1，0 ，1三种情况。只要平衡因子的绝对值大于1，则该二叉树就不是平衡的。

失衡调节

平衡二叉树的平均查找长度：log(n)

空树是平衡二叉树，在不断加点的过程中，平衡二叉树由平衡逐渐变得不平衡，如果想让平衡二叉树继续平衡，就需要根据导致不平衡的不同原因执行不同的操作，从而进行调正。

但是，无论哪种情况，首先需要处理的结点是插入新的结点后，所有祖宗结点中，距离新结点最近的节点。

一、单向右旋型(LL旋转)

原因：新插入的结点在不平衡结点的左子树的左子树中。

处理：将不平衡的结点所在的子树进行一次向右顺时针旋转的操作。

二、单向左旋型（RR旋转）

原因：新插入的结点在不平衡结点的右子树的右子树中。

处理：处理：将不平衡的结点所在的子树进行一次向左逆时针旋转的操作。

LLRR

三、先右后左型（RL旋转）

原因：新插入的结点在不平衡结点的右子树的左子树中。

处理：先围绕不平衡结点的左孩子进行逆时针左旋，在围绕不平衡结点进行顺时针右旋。

四、先左后右型（LR旋转）

原因：新插入的结点在不平衡结点的左子树的右子树中。

处理：先围绕不平衡结点的左孩子进行逆时针左旋，在围绕不平衡结点进行顺时针右旋。

BST 二叉排序（查找）树

1.定义：除了空树之外，拥有下列性质的二叉树为二叉排序树：（1）.若它的左子树不空，则左子树上所有节点的值均小于它的根结点的值。（2）.若它的右子树不空，则它的右子树上所有的节点的值均大于它的根结点的值。（3）.它的左右子树也分别为二叉排序树。

首先先看一下链表节点的定义：

struct node
{
    int data;
    node *lc;
    node *rc;
}*head;
int num[16] = {78,45,12,98,566,563,568};

2.二叉排序树的查找：

不断地和结点值相比较，如果关键字大于结点值，则指针指向右孩子继续比较，如果小于结点值，则指针指向左孩子，继续比较。如果关键字和结点值相等，则返回查找成功，否则查找不成功。

        return false;

现在用链表来实现查找：

node *findnode(int data,node *root)
{
    if(root == NULL)
        return NULL;

    if(data == root->data)
        return root;
    else if(data < root->data)
        return findnode(data,root->lc);//递归返回
    else
        return findnode(data,root->rc);
}

3.二叉排序树的建树（插入）：

创建二叉排序树和插入关键字相似，都是先查找，如果找不到就和查找的最后一个节点相比较，如果关键字大于结点值，则插到结点的右孩子的位置，如果关键字小于结点值，则插到结点的左孩子的位置。

void Insert_BST(int key,int i)
{
    if(BST[i] == 0)
    {
        BST[i] = key;
        return ;
    }
    if(BST[i] > key)
        Insert_BST(key,i*2);
    else
        Insert_BST(key,i*2+1);
}

4.二叉排序树的删除

首先需要明白，二叉排序树是有顺序的，如果随意删除一个结点，有可能会导致二叉排序树的顺序被破坏。如果要删除一个结点，则根据这个结点所在的位置进行讨论。

如果要删除的结点是叶子结点，不用顾及，尽管删除哈！！

如果要删除的结点只有一个孩子结点，左孩子也好，右孩子也好，总之把该结点删了，孩子代替父母位置就好了。

如果要删除的结点很强大，既有左孩子，又有右孩子（好强大的结点呀）。在这种情况下，就要进行中序遍历，找到要删除结点的前驱，可以进行两种操作，一种：如果要删除的节点在父节点的左子树，令删除结点的左子树为父结点的左子树，删除结点的右子树为前驱结点的右子树；如果要删除的结点在父节点的右子树上，则把删除结点的右子树放在删除节点的位置，删除结点的左子树为右子树最左边的结点的左子树。第二种是将前驱结点拿到要删除结点的位置，并且删除前驱结点。相比之下，个人认为还是第二种方法好用，(__^) 嘻嘻。

node * delnode(node *root)//接收的结点是要删除结点的父结点
{
    node *rnt;
    if(root->lc->data == data)
        //判断删除的是父结点的左孩子还是右孩子还是根节点
        rnt = root->lc;
    else if(root->rc->data == data)
        rnt = root->rc;
    else
        rnt = root;//rnt是当前要删除的结点
        node *rt;
    //采用的删除方法是第二种：
    //找到前驱结点，拿到要删除的位置，再将前驱结点删除
    if (rnt->lc)
    {
        rt = rnt->lc;
        node *qian = rt;
        while(rt->rc != NULL)
        {
            qian = rt;
            rt = rt->rc;
        }
        if(rt->lc)
        {
            qian->rc = rt->lc;
        }
        rt->rc = rnt->rc;
        rt->lc = rnt->lc;
    }
    else
    {
        rt = rnt->rc;
    }
    if(root->lc->data == data)
        root->lc = rt;
    else if(root->rc->data == data)
        root->rc = rt;
    delete(rnt);
    return root;
}

node *finddlenode(int data,node *root)
{
    if(root)
    {
        if((root->lc && root->lc->data == data) || (root->rc && root->rc->data == data) || root->data == data)
            root = delnode(root);//找到删除结点的父结点
        else//查找要删除结点的父结点
        {
            if(data < root->data)
                root->lc = finddlenode(data,root->lc);
            else
                root->rc = finddlenode(data,root->rc);
        }
    }
    return root;
}
