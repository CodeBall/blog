MySQL学习笔记（2）——数据类型及数据表操作

数据类型

1.整数类型：tinyint(8位)；smallint(16位)；mediumint(24位)；int(32位)；bigint(64位)。

2.浮点类型：float(m,d)；double(m,d)。m代表总共的位数,d代表小数的位数

3.字符类型：char(m),0-255,定长类型，不够的空格补齐；varchar(m),0-65535,不定长类型；tinytext(8)；text(16)；mediumtext(24)；longtext(32)；enum('value1','value2','value3',...),枚举类型，取决于枚举的个数，最多65535个；set('value1','value2',...),最多64个。

4.日期类型：year(默认为4为，从1970年到2069年)；time(-8385959-8385959)；date(1000.1.1-9999.12.31)；datetime(1000.1.1 0:0:0-9999.12.31 12:59:59),timestamp(时间戳，从1970.1.1 0:0:0-2037)

注意：所谓的数据优化，就是在建表的时候，选择最合适的数据类型而不是选择最大的数据类型。

数据表操作

1.创建数据表：create table [if not exists] 数据表名(字段1和字段1定义,字段2和字段2定义,..)

这种方法只是创建了一个空表，如果在建表的时候，想根据其他表的内容对新表添加数据，则可以采用create select语法结构，

create table [if not exists] table_name(字段和数据类型) select
例如：在建立产品品牌表（goods_brands）的时候，根据产品表(goods)中的产品品牌对新表进行数据填充：

create table goods_brands(
brand_id smallint unsigned primary key auto_increment,
brand_name varchar(40) not null)ENGINE=InnoDB DEFAULT CHARSET=utf8
selest brand_name from goods group by brand_name;
2.查看数据表：show tables [from 数据库名];选择数据库名即可查看当前数据库之外的其他数据库的数据表信息，否则只能查看本数据库的表信息。

3.查看数据表结构：(1).desc 表名 (2).show columns from 表名 (3).describe 表名 (4).show create table 表名

4.查看数据表的创建命令

show create table 表名;

定义字段

定义字段除了之明字段名称，字段类型之外，还包括以下几点

1.记录的唯一性：为字段添加auto_increment属性，指明该字段自动增长，默认情况下从1开始，增量为1，可以自行设置。该属性必须与接下来涉及到的主键配合使用。

2.无符号属性：unsigned指明该字段的数据类型是无符号的。

3.约束：约束是为了保证数据完整性和一致性而设定的，根据数据列的数目可以将约束分为表级约束和列级约束，表级约束即为一个约束包含了多个字段，列级约束为一个约束包含了一个字段；根据功能，约束有以下5种

a.主键约束：primary key 声明该字段为主键，每张表有且只有一个主键，而这个主键可以包含多个字段，自动为not null，声明了主键的字段值是不会重复的。

b.唯一约束：unique key声明该字段有唯一约束，字段值是不能够重复的，但是可以出现一次null，每张表可以有多个唯一约束。

c.默认约束：default指明字段的默认值，如：set varchar(6) enum('男','女','保密') default '保密'

d.非空约束：null为允许该字段出现空值，not null为不允许字段出现空值

e.外键约束：foreign key实现数据表一对一和一对多的关系，使用外键约束，外键字段和参照字段必须创建索引，如果参照字段没有索引，系统会自动创建。除此之外，外键字段和参照字段必须有相似的字段定义。而且，有外键的表称为子表，子表参照的表为父表。外键的定义格式为：foreign key (字段名) reference 父表名（参照字段）。

拓展外键约束的参照操作

更新表之后，子表是否进行相应的操作，在物理结构上可以通过外键约束的参照操作实现。

（1）cascade:从父表删除或更新记录时同步到子表相应数据中，

如：foreign key(字段名) reference 父表名（参照字段） on delete cascade on update cascade

（2）set null:从父表删除或更新数据之后，子表中的相应数据置为null，如果使用该项，必须保证子表中该字段可以置为空值。

（3）restrict:拒绝对父表的删除和更新操作

（4）no action:与rectrict相同

注意：

外键约束只有innodb这种存储引擎才能支持，其他的存储引擎是不支持这种物理的外键约束的，所以在开发过程中，一般都使用逻辑的外键约束。

修改数据表默认的存储引擎：找到MySQL配置文件my.ini，修改default-storage-engin = 存储引擎，外键约束必须依赖INNODB存储引擎，修改完成之后重启MySQL服务。

查看表中索引

show index from 表名

MySQL学习笔记（1）——概述

MySQL默认端口号：3306

MySQL语句规范

（1）登录：mysql -u用户名 -p(回车写密码（密文）|直接写密码再回车（明文）)

（2）登出：quit，\q，Ctrl+C都可以

（3）修改MySQL提示符（默认情况下时mysql）：如果在登录时修改，则：mysql -u名称 -p密码 --prompt 提示符

如果在登陆之后修改，则：prompt 新提示符

提示符可以用参数表示，比如\D代表完整的日期，\d代表当前数据库，\h代表服务器名称，\u代表当前用户，例如：prompt '\u@\h \d>'

（4）显示当前服务器版本：select version()

显示当前日期：select now()

显示当前用户：select user()

显示警告：show warning

显示编码方式：show create database 数据库名称

显示当前打开的数据库：select database()

查看当前服务器下的数据库列表：show {databases|scheam}

（5）MySQL语句规范

a.关键字与函数名称全部大写;b.数据库名称，表名称，字段名称全部小写;c.sql语句必须以分号结尾。

数据库基本操作

（1）创建数据库：create {database|scheam} [if not exists] 数据库名称 [default] character set [=] 编码方式

CREATE DATABASE db_name DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
（2）修改数据库：alter {databases|scheam} 数据库名称 [default] character set [=] 编码方式

（3）删除数据库：drop {database|scheam} [if exists] 数据库名称

Ubuntu字符界面操作--shell

使用Ubuntu已经有一段时间了,今天逛论坛发现了有关Ubuntu的各种操作指令的集合,其中有一些已经熟练应用,但还有一些没有用过,于是保存下来,方便以后查阅.

打开终端

Ctrl+Alt+T

帮助命令

（1）man工具 man为每个命令都提供了详细的信息。
man [选项参数] 命令名称 例如，man ls可显示ls命令的信息，可按下空格键显示下一屏文本信息，按“q”键退出返回到shell提示符。
man --help或者 man --usage可得到更多关于man的信息。

（2）help命令选项
help [shell内部命令] 如果help命令后面不加选项参数，则列出全部的shell内部命令。
（3）info显示命令信息 info工具显示比man工具更完整的最新GNU工具信息。
info [选项参数] 命令
info --help可查看info命令的参数

Shell基本命令

（1）查看当前路径命令pwd
pwd(Print Working Directory)，打印当前工作目录的命令。 （2）改变目录命令 cd cd [指定的目录]
上述如果cd后面没有写指定的目录，则返回到当前用户的主工作目录。进入指定目录，用户必须拥有对其执行和读权限。
（3）列举目录内容命令ls、报告当前进程快照命令ps

Shell应用技巧

1）Tab自动补全命令
例如 cd /u/lo/b，其中是按下Tab键的意思，可以将命令自动补全，上述命令将跳转到/usr/local/bin/目录。

（2）命令记忆功能

a.上下方向键 ,向上方向键向后遍历最近在该控制台下输入的命令，向下方向键向前遍历命令。
b.历史记录命令history history命令是用于读取、显示或清除命令历史记录的shell命令。
history 列出命令历史缓冲区或文件记录中记录的所有命令。

history n 可查看最近执行的n条命令。

histrory -c 可清除历史缓冲区中的命令。

c.重复执行先前命令“！”
! 引用历史缓冲区或文件中的命令

!! 重复执行前面刚执行的命令
!n 重复执行命令历史缓冲区或文件中的序号为n的命令

!-n 重复执行从当前命令位置开始倒数的第n个命令

!string 重复执行以给定的部分字符 "string"为起始字符的最近一次执行的命令

!?string[?] 重复执行包含给定的字符串"string"的最近一次执行的命令
!!string 引用前一条刚执行完的命令，将给定的字符串"string"放在该命令后，重新执行
!# 引用迄今为止已经输入的所有字符

!$ 引用前一个命令的最后一个参数

d.编辑命令行快捷键
可通过光标和Home、End等功能键对命令行进行操作，还可通过键盘快捷方式对命令行进行编辑。
Ctrl+a 光标移至行首

Ctrl+e 光标移至行尾
Ctrl+k 删除从光标到行尾的部分

Ctrl+u 删除从光标到行首的部分
Ctrl+w 删除从光标到当前单词开头的部分

Ctrl+y 插入最近删除的单词

Alt+a 将光标移到当前单词头部

Alt+e 将光标移到当前单词尾部
Alt+d 删除从光标到当前单词结尾的部分

（3）多条命令的执行

a.";"的使用
命令1；命令2 先执行命令1，不管是否出错，接下来执行命令2。例如，pwd;ls
b."&&"的使用
命令1&&命令2&&……

（4）命令别名
alias [别名]=[需要定义别名的命令] 注意，如果命令中有空格的话，需要有引号。例如，将cd /usr/local/bin命令另起别名为“bin”，用alias bin='cd /usr/local/bin'即可。

（5）管道 管道是 Linux系统中信息通信的重要方式，是先进先出的单项数据通路。利用管道线“|”可把一个命令的标准输出连接到管道。 命令1 | 命令2

（6）通配符
* 用于匹配任意字符。例如，ls *.sh 查找当前目录所有以.sh结尾的文件。

？ 用于匹配一个字符。例如，ls *.s? 查找当前目录所有以.s?结尾的文件，其中“?”是任意一个字符的意思。
[x-y] 用于匹配字符范围。例如，ls [0-9] 查找一个目录中名字里包含数字的所有文件夹。
[!x-y] 用于匹配不在字符范围。例如[!a-z]，表示除了“a-z”之间的字符
