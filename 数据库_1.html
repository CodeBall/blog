Redis

第一次使用redis进行数据缓存,现在把从安装到使用做一下笔记,方便以后看.

安装redis

通过wget下载压缩包,可以先查询最新的redis版本,然后再修改版本号下载

wget http://download.redis.io/releases/redis-3.0.7.tar.gz
解压

tar -xvf redis-3.0.7.tar.gz
进入到文件夹,执行编译

make
进入到src文件夹下,这里介绍6个可执行文件:

redis-server: Redis服务器的deamon启动程序

redis-cli: Redis命令行执行操作

redis-benchmark: Redis性能测试工具

redis-check-aof: 更新日志检测

redis-check-dump: 用于本地数据库监测

redis-sentinel: Redis实例的监控工具,通知和实例失效备援服务,是Redis集群的管理工具

回到redis-3.0.7目录下执行安装

make install
输入red并按下Teb键后会看到刚才介绍的执行文件.

修改配置文件

将源码目录下redis配置文件redis.conf拷贝到/etc/redis目录下。

修改配置项，根据需要；如果不修改，使用默认配置也可以；

运行数据库

执行redis-server命令就可以成功启动数据库了(特别注意:数据库只有成功启动了才能够进行连接,写入数据,取出数据等一系列操作.因为该数据库是运行在内存中的,所以当数据库关闭之后,存储的数据也会删除.)

PHP安装redis扩展

如果不安装扩展,就会显示系统不支持的错误,Ubuntu下使用apt-get安装扩展很方便.

sudo apt-get install php5-redis
然后重启下apache

sudo /etc/init.d/apache2 restart
ThinkPHP使用redis配置

在配置文件中写入以下配置就可以正常使用redis了.

//数据缓存设置 
'DATA_CACHE_TIME'       =>  1,             //长连接时间,REDIS_PERSISTENT为1时有效
'DATA_CACHE_PREFIX'     =>  '',            // 缓存前缀
'DATA_CACHE_TYPE'       =>  'Redis',       //数据缓存类型
'DATA_EXPIRE'           =>  0,               //数据缓存有效期(单位:秒) 0表示永久缓存
'DATA_PERSISTENT'      =>  1,               //是否长连接

//Redis缓存配置
'DATA_REDIS_HOST'            =>  '127.0.0.1', //分布式Redis,默认第一个为主服务器
'DATA_REDIS_PORT'            =>  '6379',           //端口,如果相同只填一个,用英文逗号分隔
'DATA_REDIS_AUTH'            =>  '',    //Redis auth认证(密钥中不能有逗号),如果相同只填一个,用英文逗号分隔
'REDIS_CTYPE'           => 1, //连接类型 1:普通连接 2:长连接
'REDIS_TIMEOUT'         => 0, //连接超时时间(S) 0:永不超时
'REDIS_PERSISTENT'      =>1,

pymongo操作

最近在写一个小项目,其中用到了mongodb数据库,python中使用pymongo操作mongodb数据库.所以将学习的一些知识点记录下来,方便以后查阅.

引用pymongo

import pymongo
建立连接

client = pymongo.MongoClient()
连接数据库

db = client.数据库名
db = client.words
选择集合

collect = db.集合名
collect = db.word
添加数据

添加一条数据

word = {"word_name":"beaytiful","paraphrase":"adj,美丽的,出色的,迷人的","exam_sent":"She was a beautiful girl"}
collect.insert(word)

添加多条数据

words = [{"word_name":"like","paraphrase":"vt,喜欢,愿意;vi,喜欢,希望;adv,可能","exam_sent":"What did she look like?"},{"word_name":"handsome","paraphrase":"adj,英俊的,大方的,慷慨的,健美而端庄的","exam_sent":"They will make a handsome profit on the property","exam_sent2":"He is a handsomw sheep farmer."}]
collect.insert(words)
修改数据

更新数据,为handsome单词所在的记录添加一个唯一标识

collect.update({"word_name":"handsome"},{"$set":{"word_id":1}})

更新数据,为word_id为1的数据修改名字为China

collect.update({"word_id":1},{"$set":{"word_name":"China"}})
删除数据

删除所选择的集合中的指定数据

collect.remove({"word_id":1})

删除所选择的集合中的所有数据

collect.remove()
查询数据

查看集合中的指定数据

collect.find_one({"word_id":1})

查看集合中某个数据的某些字段的值.前一个花括号写查找的数据的查找条件,后一个花括号写查询那些字段值,1为显示,0为不显示默,认_id默认显示如果不想显示则可置为0

collect.find_one({"word_id":2},{"word_name":1,"_id":0})
collect.find_one({"word_id":2},{"word_name":1,"exam_sent":1})

查看多条数据

for i in collect.find():
i

查看word_id大于1的数据的word_name,如果结果数目不止一个,采用循环打印

collect.find({"word_id":{"$gt":1}},{"word_name":1,"_id":0})

查看word_id大于等于1的数据的word_name,如果结果数目不止一个,采用循环打印

collect.find({"word_id":{"$gte":1}},{"word_name":1,"_id":0})

查看word_id小于等于1的数据的word_name,如果结果数目不止一个,采用循环打印

collect.find({"word_id":{"$lte":1}},{"word_name":1,"_id":0})

查看word_id小于3的数据的word_name,如果结果数目不止一个,采用循环打印

collect.find({"word_id":{"$lt":3}},{"word_name":1,"_id":0})

查看word_id不等于2的数据的word_name,如果结果数目不止一个,采用循环打印

collect.find({"word_id":{"$ne":2}},{"word_name":1,"_id":0})

对查询结果进行升序排序,默认为升序

for i in collect.find().sort("word_id",pymongo.ASCENDING):
i

对查询结果进行降序排序

for i in collect.find().sort("word_id",pymongo.DESCENDING):
i

对查询的结果进行多列排序

for i in collect.find({},{"word_id":1,"word_name":1,"_id":0}).sort([("word_name",pymongo.DESCENDING),("word_id",pymongo.DESCENDING)]):
i
查看一个集合中的数据总数

collect.find().count()
模糊查询

和更新数据是一样,$regex是指定该查询为模糊查询的关键,同时指定对哪一个字段进行模糊查询

Word = collect.find({"word_name":{"$regex":"l"}})

多条件模糊查询

Word = collect.find({"word_name":{"\(regex":"l"},"exam_sent":{"\)regex":"like"}})
Word = collect.find({"word_name":{"\(regex":"l"},"word_name":{"\)regex":"b"}})

MySQL学习笔记（7）——存储引擎

存储引擎在之前学习过程中接触过一点，现在再写一些涉及到的更多的知识

并发控制

当多个连接对记录进行修改时保证数据的一致性和完整性，锁系统能够实现并发控制。

锁系统

锁系统是用来进行并发控制的，有两种锁，一种是共享锁（读锁）：在同一个时间内，多个用户可以读取同一个资源，因为不会对数据进行修改，所以是可以共同使用资源的，另一种是排它锁（写锁）：在任何时候只能有一个用户写入资源，当进行写锁时会阻塞其他的读锁或者写锁操作

锁也有作用范围的大小，称为锁颗粒。比如表锁：是一种开销最小的锁，行锁：是一种开销最大的锁，支持的并发处理也是最大的。这里的开销，其实是锁的个数，表锁只对这张表加一个锁就可以了，所以开销最小，行锁要对相应的行加锁，所以开销最大。

事务处理

事务的主要作用为保证数据的完整性，事务有四大特性：原子性，一致性，隔离性，持久性；这四大特性简写为：ACID

存储引擎的定义

MySQL将数据以不同的技术存储在文件（内存）中，这种技术称为存储引擎。每一种存储引擎都是用不同的存储机制，索引技巧，锁定水平，最终提供广泛且不同的功能。

MySQL支持的存储引擎

MyISAM

InnoDB

Memory

CSV：以逗号分隔

Archive

BlackHole:黑洞引擎，写入的数据都会消失，一般用作数据复制的中继

MySQL学习笔记（6）——存储过程

MySQL命令的执行流程

对于我们输入的sql命令，MySQL引擎会先进行语法分析，确定语法正确之后对sql命令进行编译，编译成可识别的命令，然后才执行语句，并将执行的结果返回给客户端。

存储过程的定义

存储过程是sql语句和控制语句的预编译集合，以一个名称存储并作为一个存储单元处理，存储过程存储在数据库内，可以由应用程序调用执行，允许用户声明变量以及进行流程控制。

存储过程可以接收输入类型的参数，也可以接收输出类型的参数，并且可以存在多个返回值。

存储过程是预编译的，当第一次调用的时候，MySQL引擎会对其进行语法分析，编译等操作，并把结果存储在内存中。之后再调用，就直接从内存中执行。

存储过程的优点

增强sql语句的功能和灵活性：因为可以写控制语句，所以存储过程可以完成复杂的判断以及较复杂的运算

实现了较快的执行速度

减少网络流量：由原来的传递一整条语句到之后的传递参数，传递的字符量变小了。

存储过程语法结构

create
[definer = {当前用户|创建该存储过程的用户}]//如果不写则默认为当前用户
procedure 存储过程名称(参数，可以有0个到多个)
过程体
存储过程参数有3个写法，in代表这个参数的值必须是在调用存储过程时指定，存储过程中，该值不能被返回。out代表该参数的值可以被存储过程中的代码改变，可以返回。Inout表示在调用存储过程时指定，可以被改变和返回.

调用存储过程

Call 存储过程名

Call 存储过程名(参数)

修改存储过程

修改存储过程只能修改注释，内容的类型，不能够修改过程体，如果想修改过程体，则只能删除重建存储过程。

删除存储过程

delete procedure 存储过程名
创建不带参数的存储过程

创建存储过程，获取当前MySQL版本

create procedure sp1()select version();
创建带有in类型参数的存储过程

创建存储过程，用于删除特定的记录

如果存储体中有多条语句，则记得修改终止符：delimiter //，写完之后再修改回来

特别注意：参数名称和字段名称不要相同，否则会引起歧义而错删数据。

delete from tdb_goods where goods_id = id;

删除固定的记录，返回剩余的记录

create procedure removeUserAndRetuen(in p_id int unsigned,out username int unsigned)

MySQL学习笔记（5）——函数

MySQL有很多函数哦，而且函数是可以套用的~~

字符函数

字符连接：concat();例如：把名和姓进行连接输出：

select concat(first_name,last_name) as name from user;
使用指定的分隔符进行字符连接：concat_ws();

该函数第一个参数是使用的连接符，剩下的参数是需要连接的字符串，个数不限，如：

select concat('_',first_name,last_name) as name from user;
数字格式化：format();

把数字划分成千分位格式，3位为一段，第一个参数是操作的数字，第二个参数小数位数，参数不可以省略。例如：

select format(22343.5464,3);
转换成小写字母：lower();转换成大写字母：upper();只有一个参数，就是要操作的字符串。

获取左侧字符left();获取右侧字符right();有两个参数，第一个参数是操作的字符串，第二个参数是获取的长度。

获取字符串长度：length();参数就是操作的字符串

select lower('JHDSyuJH');
select upper('sdferfreHjs');
select left('ewferfr',3);
select right('ewferggrt',6);
select length('dscfd测试汉字dkj');
删除前导空格：ltrim();删除后续空格：rtrim();删除前导和后续空格：trim();

select ltrim(' 删除前导空格 ');
select rtrim(' 删除后续空格 ');
select trim(' 删除前导和后续空格 ');
trim()函数还可以删除前导和后续的其他字符

select trim(leading '?' from '???这里删除前导的问号');
select trim(trailing '?' from '??这里删除后续的问号???');
select trim(both '?' from '???这里删除前导和后续的问号???');
字符串截取：substring();该函数有三个参数，第一个是操作的字符串，第二个是从哪个字符开始截取，第三个是截取的长度，如果没有第三个参数，则默认截取到结尾。

这里需要注意的是：如果该函数配合length()函数一起使用，则对于汉字形式数据会有偏差，length()函数计算出来的长度每个汉字占两个字节，substring()函数截取的时候一个汉字占1个字节。

select substring('jjdei截取的数据ijduhu',2,8);
select substring('jjdei截取的数据ijduhu',2,5);
模式匹配：[not] like;其中%代表任意字符串（长度不限），_代表任意一个字符

字符串替换：replace();将某一个字符穿替换成另一个字符串，有三个参数，第一个是操作的字符串，第二个是替换掉的字符串，第三个是替换成的字符串

select * from user where user_name like '%亮';
select replace('??将问号??替换成/??null????','?','');
数值运算符和函数

数值运算符中，+,-,*,/自然是不用说的，下面说说其他的数值运算符

进一取整：ceil();无论小数部分有多小，都进1

舍一取整：floor();无论小数部分有多达，都舍掉

整数除法：div();不要余数，只要整除的部分

取余数：mod()或者%;获取除法运算之后余数部分，而不是小数部分哦。既可以对整数取余也可以对小数取余。

幂运算：power();

四舍五入：pound();有两个参数，第一个是操作的数字，第二个是保留的小数的位数

数字截取：truncate();有两个参数，第一个是操作的数字，第二个是保留的小数的位数，但是注意，该方法不进行四舍五入，而是直接截取

比较运算符和函数

[not] between ... And...指的是在或者不在某个区间内，这个区间是闭合的，包括边界

[not] in(),指的是在不在括号内的集合里，给出的是有限的内容点，而不是区间

Is [not] null,查看是否为空。注意：null 是空，空的字符串不是空，0也不是空

日期时间函数

当前日期和时间：now();

当前日期：curdate();

当前时间：curtime();

日期变化：date_add();可以增加日期，也可以减少日期，有两个参数，第一个参数是变化的基数，第二个参数是变化量，可以是year,month,day,week,hour,minute,second等时间单位。

日期差值：datediff();计算两个日期的差值，当然有两个参数了，在计算的时候，用第一个参数减去第二个参数。

日期格式化：date_format();也有两个参数，第一个是操作的日期，第二个是转换的格式。

select date_add(now(),interval 1 year);//将当前时间加上一年显示出来，如果是减去一年，则1变为-1
select datediff(now(),curdate());//这个结果是0，不信可以试试哦~~
select date_format(now(),'%m/%d/%y %h:%m:%s');//注意日期格式中，字母的大小写含义是不一样的
信息函数

连接ID：connection_id();

当前数据库：database();

最后插入记录的id号：last_insert_id();数据表中必须存在一个自动编号的字段

当前用户：user();

版本信息：version();

聚合函数

聚合函数只有一个返回值，这在之前就已经说过了，这里只提一下常用的聚合函数

平均值：avg();

计数：count();

最大值：max();

最小值：min();

求和：sum();

加密函数

信息摘要算法：md5();

密码算法：password();

自定义函数

语法结构：

create function 函数名(参数)//没有参数括号内可以没有内容
returns 返回类型
return 函数体
函数体可以是任意合法的SQL语句，简单的select或者insert语句，如果函数体是复合结构，则使用begin...end语句，复合结构可以包含声明，循环，控制结构。

创建不带参数的自定义函数:写一个日期类型转换的函数：

create function f1()
returns varchar(30)
return date_format(now(),'%Y年%m月%d日');
创建带有参数的自定义函数：写一个计算平均值的函数

create function f2(num1 int ,num2 int)
returns float(10,2)
return (num1 + num2)/2;
创建具有复合结构函数体的自定义函数

create function adduser(username varchar(20))
returns int
begin
insert tdb_goods(goods_name) values(username);
return last_insert_id();
end//
注意：

创建复合结构函数体时，因为内部语句用到了“;”，但是分号是语句的结束，所以会引起歧义，最好使用delimiter关键字修改mysql的结束符，如：delimiter //,即以“//”为结束符，这样就不会引起冲突了。
