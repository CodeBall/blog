MySQL学习笔记（4）——查询

为什么要存储数据呢？当然是为了查询啦~~~嘿嘿，小小无聊一下。。。接下来进入正题——查询的各种方法

where条件查询

条件查询的适用范围比较广，更新，删除都用到了where条件查询，只需要在where之后添加要查询的条件即可。

group by语句对查询结果进行分组

group by {字段名|字段位置}[asc|desc]
该语句是对查询结果进行分组，分组完成之后也可以进行排序，asc是升序（默认），desc是降序，可以按照多个字段作为分组条件，字段之间以逗号分隔。

字段位置是select 语句中查询字段所在的位置（从1开始），并不是表中字段所在的位置，不建议这么做。

group by语句有时也需要对分组进行条件确定，这时候配合使用having语句，在having之后设置分组条件，从而确定要进行分组的都有那些数据。

group by {字段名|字段位置} [asc|desc] having 分组条件
having语句进行条件选择的时候，只有两种情况，一种是条件涉及到的字段出现在select语句中（即是查询的字段之一），另一种是聚合函数（比如sum(),count()等等，这些函数之所以称为聚合函数，是因为它们永远只有一个返回值）。如果不是这两种情况，查询条件就要使用where条件查询。

order by 语句

order by {字段名|字段位置} [asc|desc]
asc是默认的升序，desc是降序。也可以根据多个字段进行排序，进行排序时，先看前面的字段，前面相同再继续往下判断。

limit限制

limit限制查询结果返回的数量，一般语法结构为：

select * from 表名 limit n
select * from 表名 limit m,n
Limit后面如果只有一个数字，即默认返回前n条数据，如果有两个数字(m,n)（数字之间以逗号分隔），则表示从第m条数据开始（从0开始计数，该计数是物理计数，和任何编号数据没有关系）,显示n条数据（包括第m条）。

查询示例

结合上述所学知识点进行小小的查询测试

select username,age,sex
from users
group by age having sex is ‘男’
order by age desc
limit 5 ;
子查询

子查询是指嵌套在查询内部，以小括号括起来的完整的查询，子查询的外层查询可以是select,insert,update,delete,set或do都是可以的。

子查询返回可以是一个标量，可以是一行，一列或者是一个子查询。

由比较运算符引发的子查询

比较运算符有：=,>,<,>=,<=,<>,!=,<=>...

例如：查询价格在所有产品平均价格之上的产品名称和价格：

select goods_name as 产品名称,goods_price as 产品价格
from goods
where goods_price >=(
select avg(goods_price) from goods);
带有any,some,all修饰的比较运算符引发的子查询

在子查询的返回结果中，有可能会有多个结果，而比较运算符只有确定和哪一个结果进行比较才可以正常运算，这时候就要用到any任何一个，some也是任何一个，all符合全部来进行修饰比较运算符。

例如：查询产品价格比任何一个超极本电脑价格都贵的产品名称和价格:

select goods_name as 产品名称,goods_price as 价格
from goods
where goods_price > any(
select goods_price from goods where goods_cate = '超级本');
由[not] in/exists引发的子查询

In 和 not in代表是否在子查询的结果里面，exists 和 not exists代表查询数据是否存在，如果存在，则返回true，不存在返回false

表的连接

一张表不可能存储所有的数据，在读取表内容的时候，我们需要根据一张表读取另一张表的内容，就用到了表的连接

使用on关键字来设定连接条件，where关键字进行结果集记录的过滤

内连接：inner join，仅显示符合连接条件的记录

左外连接：left join，显示左表中的全部和右表中的符合条件的数据，如果左表中数据在右表中没有符合条件的字段，则显示为null

右外链接：right join，和左外连接相反

无限级分类的表设计

其实也是很简单的，表结构中除了原有的id,name等字段之外，再添加一个字段，那就是该分类的父类id。例如：

CREATE TABLE tdb_goods_types(
type_id SMALLINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
type_name VARCHAR(20) NOT NULL,
parent_id SMALLINT UNSIGNED NOT NULL DEFAULT 0
);
自身连接进行无限级分类的查找，找出每一个分类的自身id，自身名称，父类名称，这里将一张表想象成两张表进行左外连接（为什么选用左外连接呢？因为顶级分类是没有父级分类的，但是我们不能够不查询出来，所以采用左外连接，将子表设为左表）

Select s.type_id as 自身id,s.type_name as 自身名称,p.type_name as 父类名称
From tdb_goods_types as s left join tdb_goods_types as p on s.parent_id = p.type_id;

MySQL学习笔记（3）——修改数据表及写数据

操作字段

添加单个字段：

alter table 表名 add [column] 字段名称 字段定义 [first|after 字段名称]
first指明将该字段添加到表定义的最前端，after指明将该字段添加到某个字段的后面，默认情况下是添加到最后。

添加多个字段：

alter table 表名 add column
添加多个字段是不能够指定位置的，很显然，没法指定。。。

删除一个字段：

alter table 表名 drop 字段名称
删除多个字段：

alter table 表名 drop 字段名称,drop 字段名称
删除字段的同时添加字段：

alter table 表名 drop 字段名称,drop 字段名称,add 字段名称 字段定义
操作约束

添加主键约束：

alter table 表名 [constraint [约束名称]] primary key 约束类型
添加唯一约束：

alter table 表名 [constraint [约束名称]] unique key 约束类型
添加外键约束：

alter table 表名 [constraint [约束名称]] foreign key (外键字段名) references 从表名（字段）
添加|删除默认约束：

alter table 表名 later [column] 字段名 {set default 默认值|drop default}
删除主键约束：

alter table 表名 drop primary key
删除唯一约束：

alter table 表名 drop {index|key} 约束名
删除外键约束：

alter table 表名 drop foreign key 约束名
操作数据表

修改字段定义：

alter table 表名 modify [column] 字段名 字段定义 [first|after 字段名]
修改字段名称：

alter table 表名 change [column] 旧字段名称 新字段名称 字段定义 [first|after 字段名]
修改数据表名称：

alter table 表名 rename [to|as] 新名称
rename table 表名 to 新名 [,表名1 to 新名1]...
插入数据

方法一

插入多条数据

insert [into] 表名 [(字段名称)] values(字段值)
字段名称可写可不写，如果不写，则字段值必须包含所有字段的值，如果字段是自动编号，则可以写为null或者default；如果写字段名，则按照给定的字段顺序依次填入相应的字段值，字段值可以是表达式，如果该字段有默认值，也可以是默认值（default）

插入多条数据

insert [into] 表名 [(字段名称)] values(字段值1),(字段值2),...
如果插入多条数据，则必须有相同个字段，即不能第一条数据有3个值，第二条数据有5个值。

方法二

insert 表名 set 字段名1 = '字段值1',字段名2 = '字段值2',...
这种方法只能插入一条语句，可以进行子查询，具体来说是比较运算符=引发的子查询

方法三

insert [into] 表名 [(字段名称)] select 选择语句
这种方法借助查询语句来从另外一张表中查找到相应数据插入到该表中

比如在产品表中查询产品的分类，将查询的结果写入到分类表中

insert goods_cates (cate_name) select
goods_cate from goods group by goods_cate;
修改数据

单表修改

update 表名 set 字段名=’更新的该字段的值’[,字段名1=’更新的该字段的值1’,...] [where 更新条件]
如果不写更新条件，则所有的数据都将更新

多表修改

多表更新涉及到表与表之间的连接，根据一张表的内容更新另一张表的内容

例如：根据产品类型表(goods_cate)更新产品表(goods)中的产品类型

Update goods inner join goods_cate on goods_cate = cate_name set goods_cate = cate_id;
删除数据

单表删除

Delete from 表名 [where 删除条件]
如果不写删除条件，则该表中所有的数据都会被删除。

多表删除

所谓的多表删除，就是通过多张表的连接来确定删除的内容。

例如：在一张表(goods)中，有很多重复的记录，如果想删除重复的记录，不可能找到它们的id一个一个的删除，这就用到了多表删除。把这张表和由重复数据组成的虚拟的表进行连接，进而进行删除操作。

Delete t1 from goods as t1 left join(
select goods_id,goods_name from goods group by goods_name having count(goods_name)>=2) as t2
on t1.goods_name = t2.goods_name
where t1.goods_id > t2.goods_id;

MySQL学习笔记（2）——数据类型及数据表操作

数据类型

1.整数类型：tinyint(8位)；smallint(16位)；mediumint(24位)；int(32位)；bigint(64位)。

2.浮点类型：float(m,d)；double(m,d)。m代表总共的位数,d代表小数的位数

3.字符类型：char(m),0-255,定长类型，不够的空格补齐；varchar(m),0-65535,不定长类型；tinytext(8)；text(16)；mediumtext(24)；longtext(32)；enum('value1','value2','value3',...),枚举类型，取决于枚举的个数，最多65535个；set('value1','value2',...),最多64个。

4.日期类型：year(默认为4为，从1970年到2069年)；time(-8385959-8385959)；date(1000.1.1-9999.12.31)；datetime(1000.1.1 0:0:0-9999.12.31 12:59:59),timestamp(时间戳，从1970.1.1 0:0:0-2037)

注意：所谓的数据优化，就是在建表的时候，选择最合适的数据类型而不是选择最大的数据类型。

数据表操作

1.创建数据表：create table [if not exists] 数据表名(字段1和字段1定义,字段2和字段2定义,..)

这种方法只是创建了一个空表，如果在建表的时候，想根据其他表的内容对新表添加数据，则可以采用create select语法结构，

create table [if not exists] table_name(字段和数据类型) select
例如：在建立产品品牌表（goods_brands）的时候，根据产品表(goods)中的产品品牌对新表进行数据填充：

create table goods_brands(
brand_id smallint unsigned primary key auto_increment,
brand_name varchar(40) not null)ENGINE=InnoDB DEFAULT CHARSET=utf8
selest brand_name from goods group by brand_name;
2.查看数据表：show tables [from 数据库名];选择数据库名即可查看当前数据库之外的其他数据库的数据表信息，否则只能查看本数据库的表信息。

3.查看数据表结构：(1).desc 表名 (2).show columns from 表名 (3).describe 表名 (4).show create table 表名

4.查看数据表的创建命令

show create table 表名;

定义字段

定义字段除了之明字段名称，字段类型之外，还包括以下几点

1.记录的唯一性：为字段添加auto_increment属性，指明该字段自动增长，默认情况下从1开始，增量为1，可以自行设置。该属性必须与接下来涉及到的主键配合使用。

2.无符号属性：unsigned指明该字段的数据类型是无符号的。

3.约束：约束是为了保证数据完整性和一致性而设定的，根据数据列的数目可以将约束分为表级约束和列级约束，表级约束即为一个约束包含了多个字段，列级约束为一个约束包含了一个字段；根据功能，约束有以下5种

a.主键约束：primary key 声明该字段为主键，每张表有且只有一个主键，而这个主键可以包含多个字段，自动为not null，声明了主键的字段值是不会重复的。

b.唯一约束：unique key声明该字段有唯一约束，字段值是不能够重复的，但是可以出现一次null，每张表可以有多个唯一约束。

c.默认约束：default指明字段的默认值，如：set varchar(6) enum('男','女','保密') default '保密'

d.非空约束：null为允许该字段出现空值，not null为不允许字段出现空值

e.外键约束：foreign key实现数据表一对一和一对多的关系，使用外键约束，外键字段和参照字段必须创建索引，如果参照字段没有索引，系统会自动创建。除此之外，外键字段和参照字段必须有相似的字段定义。而且，有外键的表称为子表，子表参照的表为父表。外键的定义格式为：foreign key (字段名) reference 父表名（参照字段）。

拓展外键约束的参照操作

更新表之后，子表是否进行相应的操作，在物理结构上可以通过外键约束的参照操作实现。

（1）cascade:从父表删除或更新记录时同步到子表相应数据中，

如：foreign key(字段名) reference 父表名（参照字段） on delete cascade on update cascade

（2）set null:从父表删除或更新数据之后，子表中的相应数据置为null，如果使用该项，必须保证子表中该字段可以置为空值。

（3）restrict:拒绝对父表的删除和更新操作

（4）no action:与rectrict相同

注意：

外键约束只有innodb这种存储引擎才能支持，其他的存储引擎是不支持这种物理的外键约束的，所以在开发过程中，一般都使用逻辑的外键约束。

修改数据表默认的存储引擎：找到MySQL配置文件my.ini，修改default-storage-engin = 存储引擎，外键约束必须依赖INNODB存储引擎，修改完成之后重启MySQL服务。

查看表中索引

show index from 表名

MySQL学习笔记（1）——概述

MySQL默认端口号：3306

MySQL语句规范

（1）登录：mysql -u用户名 -p(回车写密码（密文）|直接写密码再回车（明文）)

（2）登出：quit，\q，Ctrl+C都可以

（3）修改MySQL提示符（默认情况下时mysql）：如果在登录时修改，则：mysql -u名称 -p密码 --prompt 提示符

如果在登陆之后修改，则：prompt 新提示符

提示符可以用参数表示，比如\D代表完整的日期，\d代表当前数据库，\h代表服务器名称，\u代表当前用户，例如：prompt '\u@\h \d>'

（4）显示当前服务器版本：select version()

显示当前日期：select now()

显示当前用户：select user()

显示警告：show warning

显示编码方式：show create database 数据库名称

显示当前打开的数据库：select database()

查看当前服务器下的数据库列表：show {databases|scheam}

（5）MySQL语句规范

a.关键字与函数名称全部大写;b.数据库名称，表名称，字段名称全部小写;c.sql语句必须以分号结尾。

数据库基本操作

（1）创建数据库：create {database|scheam} [if not exists] 数据库名称 [default] character set [=] 编码方式

CREATE DATABASE db_name DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
（2）修改数据库：alter {databases|scheam} 数据库名称 [default] character set [=] 编码方式

（3）删除数据库：drop {database|scheam} [if exists] 数据库名称
