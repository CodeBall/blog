哈希表

概念

为了方便查找，在存储位置和关键字之间建立一个确定的关系，当查找关键字的时候，根据建立的关系对存储位置进行访问，有效地减少了多次比较带来的时间的浪费。这个关系称为哈希函数，这种思想称为哈希思想。

哈希函数是一个映像，可以灵活得定义。但是，对于不同的关键字，经过相同的哈希函数的处理后，有可能得到相同的地址，这种现象称为冲突，并且，一般情况下，冲突只能尽可能的减少，而不能完全避免。

哈希函数的构造方法

（1）直接定址法

去关键字或关键字的某个线性函数作为哈希函数，由此计算哈希地址的方法。

H(key) = key或H(key) = a*key + b(a,b为常数)。

（2）数字分析法

假设关键字是R进制数，并且哈希表中可能出现的关键字都是事先知道的，则可取关键字的若干数位组成哈希地址。

（3）平方取中法

取关键字平方后的中间几位作为哈希地址。

（4）折叠法

将关键字分割成位数相同的几部分（最后一部分的位数可以不同），然后取这几部分的叠加和作为哈希地址的方法。

适用于关键字位数很多，而且关键字中每一位上数字分布大致均匀。

（5）除留余数法（最简单，最常用）

取关键字被某个不大于哈希表表长m的数p除后所得余数为哈希地址。H（key） = key % p;

在选取p的时候，一般选质数或者不包含小于20的质因数的和数。

（6）随机数法

选择一个随机函数，取关键字的随机函数值为它的哈希地址。

冲突的处理

（1）开放定址法

Hi = （H（key）+di）%m;i = 1,2,...,k（k <= m-1）

H（key）为哈希函数，m为哈希表表长，di为递增序列

di有三种取法

线性探测再散列：di = 1,2,3,...,m-1;保证只要哈希表未填满，总能找到一个不发生冲突的地址

二次探测再散列：di = 11,-1-1,22,-2-2,33,...,kk,-k-k;只有在哈希表长满足4j+3一形式的素数时才会将冲突处理好。

伪随机探测再散列：di = 伪随机数序列。

二次聚集 在处理冲突过程中发生的两个第一个哈希地址不同的记录争夺同一个后记哈希地址的现象，即在处理同义词的冲突过程中又添加了非同义词的冲突。

（2）再哈希表

Hi = R*Hi(key)，i = 1,2,3。。。k;

R*Hi(key)均是不相同的哈希函数，即在同义词产生地址冲突时计算另一个哈希函数，直到不产生地址冲突为止。

（3）链地址法（方便，常用）

将所有关键字为同义词的记录存储在同一线性链表中。假设某哈希函数产生的哈希地址在区间【0,m-1】中，则建立一个头链表。每个链结的初始状态都是空指针，当有关键字对应一个链结的时候，在该链结后加关键字，如图

（4）建立一个公共溢出区

一个基本表，每个分量存放一个记录，另设一个溢出表，所有关键字和基本表中关键字为同义词的记录，不管哈希地址是什么，一旦发生冲突，都填入溢出表。

哈希表的查找

给定K值，根据建表时设定的哈希函数求得哈希地址，若表中此位置上没有记录，则查找不成功，否则比较关键字，如果给定的K和表中记录相等，则查找成功；否则根据建表时设定的冲突处理的方法进行下一个地址的查找，直到哈希表的某个位置为空或者找到为止。

查找过程中需和给定值进行比较的关键字的个数取决于三个因素：哈希函数，处理冲突的方法和哈希表的装填因子。

装填因子：a = （表中填入的记录数）/（哈希表的长度）。标志哈希表的装满程度。

平衡二叉树

基本概念

平衡二叉树又称AVL树，它的左子树和右子树都是平衡二叉树，并且左子树和右子树的深度之差的绝对值不超过1.

平衡因子：该结点的左子树的深度减去该结点的右子树的深度，所以平衡二叉树上的所有结点的平衡因子只可能是-1，0 ，1三种情况。只要平衡因子的绝对值大于1，则该二叉树就不是平衡的。

失衡调节

平衡二叉树的平均查找长度：log(n)

空树是平衡二叉树，在不断加点的过程中，平衡二叉树由平衡逐渐变得不平衡，如果想让平衡二叉树继续平衡，就需要根据导致不平衡的不同原因执行不同的操作，从而进行调正。

但是，无论哪种情况，首先需要处理的结点是插入新的结点后，所有祖宗结点中，距离新结点最近的节点。

一、单向右旋型(LL旋转)

原因：新插入的结点在不平衡结点的左子树的左子树中。

处理：将不平衡的结点所在的子树进行一次向右顺时针旋转的操作。

二、单向左旋型（RR旋转）

原因：新插入的结点在不平衡结点的右子树的右子树中。

处理：处理：将不平衡的结点所在的子树进行一次向左逆时针旋转的操作。

LLRR

三、先右后左型（RL旋转）

原因：新插入的结点在不平衡结点的右子树的左子树中。

处理：先围绕不平衡结点的左孩子进行逆时针左旋，在围绕不平衡结点进行顺时针右旋。

四、先左后右型（LR旋转）

原因：新插入的结点在不平衡结点的左子树的右子树中。

处理：先围绕不平衡结点的左孩子进行逆时针左旋，在围绕不平衡结点进行顺时针右旋。

BST 二叉排序（查找）树

1.定义：除了空树之外，拥有下列性质的二叉树为二叉排序树：（1）.若它的左子树不空，则左子树上所有节点的值均小于它的根结点的值。（2）.若它的右子树不空，则它的右子树上所有的节点的值均大于它的根结点的值。（3）.它的左右子树也分别为二叉排序树。

首先先看一下链表节点的定义：

struct node
{
    int data;
    node *lc;
    node *rc;
}*head;
int num[16] = {78,45,12,98,566,563,568};

2.二叉排序树的查找：

不断地和结点值相比较，如果关键字大于结点值，则指针指向右孩子继续比较，如果小于结点值，则指针指向左孩子，继续比较。如果关键字和结点值相等，则返回查找成功，否则查找不成功。

bool Search_BST(int key,int i)
{
    if(BST[i] == key)
        return true;
    if(i > (pow(2,15/2+1))-1)//这个地方一定要注意，因为数组建树是满二叉树
        //所以判断没有找到的一定要注意区间范围
        return false;
    if(BST[i] > key)
        Insert_BST(key,i*2);
    else
        Insert_BST(key,i*2+1);
}

node *findnode(int data,node *root)

3.二叉排序树的建树（插入）：

创建二叉排序树和插入关键字相似，都是先查找，如果找不到就和查找的最后一个节点相比较，如果关键字大于结点值，则插到结点的右孩子的位置，如果关键字小于结点值，则插到结点的左孩子的位置。

//在查找插入的过程中建树，注意这种用数组表示的方法

4.二叉排序树的删除

首先需要明白，二叉排序树是有顺序的，如果随意删除一个结点，有可能会导致二叉排序树的顺序被破坏。如果要删除一个结点，则根据这个结点所在的位置进行讨论。

如果要删除的结点是叶子结点，不用顾及，尽管删除哈！！

如果要删除的结点只有一个孩子结点，左孩子也好，右孩子也好，总之把该结点删了，孩子代替父母位置就好了。

如果要删除的结点很强大，既有左孩子，又有右孩子（好强大的结点呀）。在这种情况下，就要进行中序遍历，找到要删除结点的前驱，可以进行两种操作，一种：如果要删除的节点在父节点的左子树，令删除结点的左子树为父结点的左子树，删除结点的右子树为前驱结点的右子树；如果要删除的结点在父节点的右子树上，则把删除结点的右子树放在删除节点的位置，删除结点的左子树为右子树最左边的结点的左子树。第二种是将前驱结点拿到要删除结点的位置，并且删除前驱结点。相比之下，个人认为还是第二种方法好用，(__^) 嘻嘻。

node * delnode(node *root)//接收的结点是要删除结点的父结点
{
    node *rnt;
    if(root->lc->data == data)
        //判断删除的是父结点的左孩子还是右孩子还是根节点
        rnt = root->lc;
    else if(root->rc->data == data)
        rnt = root->rc;
    else
        rnt = root;//rnt是当前要删除的结点
        node *rt;
    //采用的删除方法是第二种：
    //找到前驱结点，拿到要删除的位置，再将前驱结点删除
    if (rnt->lc)
    {
        rt = rnt->lc;
        node *qian = rt;
        while(rt->rc != NULL)
        {
            qian = rt;
            rt = rt->rc;
        }
        if(rt->lc)
        {
            qian->rc = rt->lc;
        }
        rt->rc = rnt->rc;
        rt->lc = rnt->lc;
    }
    else
    {
        rt = rnt->rc;
    }
    if(root->lc->data == data)
        root->lc = rt;
    else if(root->rc->data == data)
        root->rc = rt;
    delete(rnt);
    return root;
}

node *finddlenode(int data,node *root)
{
    if(root)
    {
        if((root->lc && root->lc->data == data) || (root->rc && root->rc->data == data) || root->data == data)
            root = delnode(root);//找到删除结点的父结点
        else//查找要删除结点的父结点

                root->lc = finddlenode(data,root->lc);
            else
                root->rc = finddlenode(data,root->rc);
        }
    }

}

越担心失败越容易失败

失败，不是个讨人喜欢的词呦。人们谈起它，唯恐避之不及，赶紧说：呸呸！人们遇上它，叫苦不迭，连声哀叹自己"命不好"。可是，会不会有许多失败的因子已经悄然潜伏在我们的潜意识中呢？

作为一名临近毕业的大学生，我是有多担心自己的工作啊！每天上课、自习、考证、实习。似乎每件事都与未来有关。可分明一边还在焦虑着：现在就业那么难，未来是否会有一个岗位是属于我的？求职失败了我该肿么办？ 难道我这真的是"未老先衰"的节奏啊……

别急，听我往下说。事实上，还有许多人如同我一样，提前预想到自己可能的"下场"，他们或许是焦躁难耐的大学生；或许是进退维谷的创业者；或许是压力山大的白领一族；更有可能是浮躁社会里的你你我我。担心失败的大有人在，单听他们的诉说，就让人慌了阵脚。我不禁想到：担心什么来什么。我们选择的是不安，结局亦是不安；我们选择把失败作为假想敌。结局就不言而喻，亦是失败！

哎呀糟糕！开篇我就讲了这么多"失败"，接下来的运气肯定不会好了……事实果真是这样吗？听起来好像失败来得如此简单。有人说，这世界上存在一种"吸引力"法则，我们可以这样理解：你生命中所发生的一切，都是你吸引来的。这样说你会不会感觉心安呢？因为若是这样，成功也变得相当简单，你时常想着它念着它，没事把它放心里意淫千百次，成功它就真的追着你来了！

成功与失败从来都不是耍大牌的大腕，而是挥之即来呼之即去的小喽啰！

一、担心什么来什么（心理暗示的力量）

澳大利亚一名电视工作者 朗达·拜恩，于2008年出版了一本名为 The Secret（《秘密》）的书，曾引起巨大反响。这本书最核心的内容就是阐述了一个法则，即 "吸引力法则"，大意是说"心中所想之事越发强烈，似乎就越容易实现"，这无疑会让人们联想到永远抱有巨奖梦的彩民们。

这本书阐述的"吸引力法则"基本原理是这样的：人类所有的思维活动，都会产生某种特定的频率（脑电波），而这种频率就好比杜鹃用于求爱的信号、蝙蝠用来探路的超声波。它会吸引同样的频率，引发共振，从而将我们思维活动中所涉及到的任何事物吸引到我们的面前。

据说有些彩民将自己买彩票中奖的幸运归结为"自己的信念足够强大"，好了，先别慨叹"心想事成"的力量之强大了。不管它是"吸引力法则"也好，"心理暗示"也好，作为相反的例子，我们越是担心失败，脑中的磁场就会充分调动周围的资源，将那些与失败相关的，充满负面思维的事物吸引至我们眼前。然后我们面对着它们，就好像面对着久久未见的眼熟人儿，既熟悉又陌生。因为担心失败，大脑就更关注失败，会想到很多和失败相关的事情，如失败后的各种后果。因为大脑的思考范围有限，这么多心思放在"担心"上，就没有足够能量来考虑解决问题的办法，自然没那么多精力去做或许能真正"促进成功"的事情，失败几率自然更大。

二、墨菲定律的启示（心理倾向的作用）

"墨菲定律"（Murphy's Law）：事情往往会向你所想到的不好的方向发展，只要有这个可能性。比如你衣袋里有两把钥匙，一把是你房间的，一把是汽车的；如果你现在想拿出车钥匙，会发生什么？是的，你往往是拿出了房间钥匙。这就是著名的"墨菲定律"，也叫"倒霉蛋定律"。根据它，我们得知：

  1. 任何事都没有表面看起来那么简单；

  2. 所有的事都会比你预计的时间长；

  3. 会出错的事总会出错；

  4. 如果你担心某种情况发生，那么它就更有可能发生。

我们都有这样的体会，如果在街上准备拦一辆车去赴一个时间紧迫的约会，你会发现街上所有的出租车不是有客就是根本不搭理你，而当你不需要租车的时候，却发现有很多空车在你周围游弋，只待你的一扬手，车随时就停在你的面前。如果一个月前在浴室打碎镜子，尽管仔细检查和冲刷，也不敢光着脚走路，等过了一段时间确定没有危险了，不幸的事还是照样发生，你还是被碎玻璃扎了脚。

在运动场上，我们也不乏看见这样的例子。美国射手埃蒙斯，在2004年雅典奥运会男子步枪三姿决赛，前九枪领先对手3环之多，最后一枪鬼使神差地把子弹打到了别人的靶子上。2008年北京奥运会男子50米步枪3x40决赛，埃蒙斯在倒数第二轮领先将近4环，金牌几乎唾手可得的情况下，重演了雅典的严重失误，最后一轮仅打出了4.4环。2012年伦敦奥运会射击项目的最后一个比赛日，男子50米步枪三姿决赛，埃蒙斯再次折在"最后一枪"。

也许他与成功只相差一步。但心里若对比赛结果有那么一点点介意，便草木皆兵了罢。当我们在惋惜埃蒙斯的三度梦碎的时候，是否会从中得到一些启示呢？

三、人的"自我验证"机制（心理预期的作用）

"安全感"是大多数人所追求的。我们去幻想失败，也许没有其他的想法，只是为了证实自己心中所想，以达到心理预期，也就是所谓的安全感。我们在接受外界信息的时候往往都倾向于接受能够保持和强化自我概念的信息，来增强确定性，消除不确定性所带来的心理不安感。

根据swann的理论，有积极自我概念的人，认为自己会成功的人，他会选择营造验证自我的社会环境：（1）他会选择和成功的人交往；（2）他会在别人面前展示成功者的面貌；（3）采用能够验证自己成功的交往策略。而且会对现实信息进去主观歪曲：（1）选择性地注意那些成功的事迹；（2）选择性的对事情解释，失败的事情都能被解释为成功的事。

用个例子来说明下：我准备了很久的考试，却每次都是考到我没有背的那些题目呢？原因可以解释为：

1、反正会漏背，背不完，我就挑选几道可能考的题目（消极的策略）

2、实际上有很多次刚好考到他背的题目，但他却总是记得那些漏背的考试，因为这是让人沮丧的情况所以让人印象深刻（选择性注意）

3、实际上不是漏背，只是没背熟，他为了证明自己的倒霉而不是记忆力不好，解释为漏背了（选择性解释）

所以，为什么越担心失败就越容易失败呢？因为这种担心失败的不积极的自我概念，会让我们展现出失败者的面貌，进而激发我们采取退缩、回避等消极策略。在面对问题时，消极策略让我们不能倾尽全力去付出，一旦结果不如人意将再次强化自己会失败的自我概念。就算在整件事当中有些事是成功的，但是我们往往选择性地忘记成功，反而牢牢地记住自己的失败，并深信不疑，觉得自己的成功是侥幸，失败是很经常和容易发生的（选择性歪曲）。

说到这里，我想大家都知道了这个问题的答案了。原来我们心心念念的那个"情人"是"失败"呀！什么，你并不爱它？那么，为什么不从容地向成功挥一挥手，展示我们对它的热切渴望？成功，从来都不是大腕，它就是你身边一打杂儿的伙计，你向它招一招手，它就屁颠屁颠儿地跟来了，当然你得诚心诚意呼唤它，用100%努力去召唤它，这样它定不会背叛你。

PS. 这一招在 Hard 难度以上的现实世界效果会打折扣。但起码是把成功拉向自己，而不是傲娇地一把把它推开。
